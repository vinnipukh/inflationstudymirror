# Plan 02-01: Add Falcon app/resources and response serialization

<objective>
Create the thin Falcon adapter layer for the dashboard backend. This plan adds dependency metadata, a side-effect-free API package, the Falcon `create_app()` factory, route registration for all locked endpoints, shared response envelopes, JSON-native serialization, cached inventory/filter helpers, and happy-path endpoint resources that reuse the Phase 1 CSV adapter and application use cases. The API layer must not import Streamlit or duplicate CSV/domain logic. Schema push gate result: no ORM/schema files are relevant to this Python/Falcon/CSV phase, so do not add any schema migration or schema push task.
</objective>

## Artifacts this phase produces

- `inflation_dashboard/api/__init__.py` — side-effect-free API package marker.
- `inflation_dashboard/api/falcon_app.py` — `create_app()` Falcon WSGI app factory registering `/api/health`, `/api/inventory`, `/api/history`, `/api/retailer-averages`, `/api/movers`, and `/api/coverage`.
- `inflation_dashboard/api/serialization.py` — API response helpers: `to_json_value`, `json_safe`, `records_from_frame`, `json_safe_mapping`, `envelope`, and `error_envelope`.
- `inflation_dashboard/api/filters.py` — API query/loading helpers: `ApiFilterError`, `ParsedFilters`, `get_inventory`, `clear_inventory_cache`, `parse_bool_param`, `parse_common_filters`, `load_filtered_history`, and metadata helpers for selected inventory/history/skipped counts.
- `inflation_dashboard/api/resources.py` — Falcon resources: `HealthResource`, `InventoryResource`, `HistoryResource`, `RetailerAveragesResource`, `MoversResource`, and `CoverageResource`.
- `pyproject.toml` dependency entry for Falcon, if the project chooses `pyproject.toml` as the runtime metadata source.
- `requirements.txt` dependency entry `falcon`, if the project keeps this manifest as the install surface for the current workflow.
- Endpoint paths created in this plan: `/api/health`, `/api/inventory`, `/api/history`, `/api/retailer-averages`, `/api/movers`, `/api/coverage`.
- Query parameters accepted by the API layer: repeated `retailer`, `start_date`, `end_date`, `max_files`, `all_history`, `product_name`, `product_retailer`, `aggregation`, `scope_retailer`, `limit`, and `category_limit`.
