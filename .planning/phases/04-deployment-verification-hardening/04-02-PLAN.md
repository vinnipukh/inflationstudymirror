# Plan 04-02: Combined Smoke Test & Test Documentation

<objective>
Create a combined full-stack smoke test that starts the Falcon API in-process via TestClient, validates all API endpoints, then verifies the Streamlit frontend API client can correctly consume those endpoints.
</objective>

<tasks>

### Task 04-02-01: Read existing verifier scripts

Read both existing scripts to adopt the same patterns, assertion style, and exit code convention.

**Files to read:**
- `scripts/verify_falcon_api.py`
- `scripts/verify_streamlit_api_frontend.py`

### Task 04-02-02: Create combined full-stack smoke test

Write `scripts/verify_full_stack.py` that:

1. **Runs all Falcon API endpoint smoke checks** (same as verify_falcon_api.py)
2. **Runs all Streamlit API client boundary checks** (same as verify_streamlit_api_frontend.py)
3. **Runs cross-layer integration checks:**
   - Start in-process Falcon TestClient
   - Use `inflation_dashboard.frontend.api_client` to call each endpoint through the test client
   - Verify response envelopes are parseable by the frontend
   - Verify error handling works end-to-end (invalid filter → API error → client raises ApiClientError)
4. **Exit 0 on all pass, exit 1 on any failure**
5. **Use the same bounded loading** (max_files=1) to avoid full dataset scans

**Important:** Must not start a persistent server — uses Falcon's in-process TestClient.
Streamlit frontend API client must call directly through the TestClient, not via HTTP.

### Task 04-02-03: Update TESTING.md with test run commands

Add sections for all three verification modes:
```bash
# Run Falcon API smoke test
uv run python scripts/verify_falcon_api.py

# Run Streamlit frontend API client test
uv run python scripts/verify_streamlit_api_frontend.py

# Run combined full-stack smoke test
uv run python scripts/verify_full_stack.py

# Run all verifiers at once
uv run python scripts/verify_full_stack.py
```

### Task 04-02-04: Run and verify the combined smoke test

```bash

```
Confirm exit 0 and expected output.
