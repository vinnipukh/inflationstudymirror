# Plan 03-01: Add API client layer and wire sidebar/filter flows to API calls

<objective>
Introduce the strict Streamlit-to-Falcon HTTP boundary and move the dashboard sidebar/inventory/filter setup off local CSV discovery. The API client should use explicit timeouts: short requests such as `/api/health` and `/api/inventory` around 10 seconds, and data requests around 60 seconds because endpoints are backed by bounded CSV loading. Sidebar defaults should preserve current behavior: use `DEFAULT_RETAILERS` when available in the inventory response, use the last 60 days as the default date window, and use `DEFAULT_MAX_FILES_PER_RETAILER` for the file cap, implemented without importing CSV adapter constants. Streamlit must expose a configurable Falcon API base URL defaulting to `http://localhost:8000`, call `/api/inventory` for retailer/date metadata, build shared common query params for later tab endpoint calls, validate the Falcon `{data, meta, errors}` envelope, surface client/API errors with `st.error`, and avoid any direct CSV fallback. This plan intentionally leaves the full four-tab data refactor to 03-02, but it must remove the local inventory scan path and establish verifier coverage for the new client boundary, repeated retailer params, and forbidden local inventory imports/calls.
</objective>

## Artifacts this phase produces

- `inflation_dashboard/frontend/__init__.py` — side-effect-free frontend package marker if a new package is created.
- `inflation_dashboard/frontend/api_client.py` — Streamlit-free Falcon HTTP client boundary with `ApiClientError`, `DashboardFilters`, `ApiEnvelope`, `DEFAULT_API_BASE_URL`, `FRONTEND_DEFAULT_RETAILERS`, `FRONTEND_DEFAULT_MAX_FILES_PER_RETAILER`, `SHORT_TIMEOUT_SECONDS`, `DATA_TIMEOUT_SECONDS`, `normalize_api_base_url`, `build_common_params`, `fetch_endpoint`, `fetch_health`, `fetch_inventory`, and endpoint stubs/wrappers needed by 03-02.
- `streamlit_app.py` API-backed sidebar/client integration: `st.sidebar.text_input` for Falcon API base URL, cached inventory fetch wrapper, API error display helper, inventory payload parsing, default retailer/date/cap construction, and a shared `DashboardFilters` instance passed to tab renderers.
- `scripts/verify_streamlit_api_frontend.py` — focused frontend source/client verifier covering forbidden Streamlit data-boundary imports, base URL control, repeated retailer params, envelope validation, client error behavior, and 03-01 sidebar/inventory expectations.
- Query parameter representation produced by the frontend: repeated `retailer`, `start_date`, `end_date`, `max_files`, and `all_history`.
- UI constants introduced in the frontend boundary: default API URL `http://localhost:8000`, default retailers `Markets / Gurmar`, `ClothingStores / Vakko`, `HomeGoods`, default max files `45`, short timeout about `10`, and data timeout about `60`.
