# Plan 03-02: Refactor all four tabs to use API payloads and preserve existing UX behavior

<objective>
Complete the Streamlit API frontend refactor by moving Product explorer, Retailer averages, Price movers, and Coverage overview tab data to Falcon endpoint payloads. This plan depends on 03-01's API client/sidebar/filter boundary. It removes the remaining shared local `history, skipped = load_price_history(...)` dependency and direct `inflation_dashboard.application.use_cases` calls from `streamlit_app.py`, while preserving chart/table UX, search/autocorrection controls, tab labels, bounded loading filters, empty-state behavior, and visible error handling. Chart spec imports from `inflation_dashboard.application.chart_specs` may remain because they are rendering metadata.
</objective>

## Artifacts this phase produces

- `streamlit_app.py` refactored tab renderers:
  - `render_product_explorer(api_base_url, filters, selected_retailers)` or equivalent, using bounded `/api/history` for product options and product-specific `/api/history` for chart/summary.
  - `render_retailer_average(api_base_url, filters, retailer_options)` or equivalent, using `/api/retailer-averages` with `aggregation`.
  - `render_price_movers(api_base_url, filters, selected_retailers)` or equivalent, using `/api/movers` with `scope_retailer` and `limit`.
  - `render_overview(api_base_url, filters)` or equivalent, using `/api/coverage` with `category_limit=20`.
- `streamlit_app.py` API payload helpers if needed: `records_to_frame`, `parse_date_column`, `safe_table_columns`, `display_api_warnings`, `render_api_error`, `empty_payload_info`, or equivalent helpers for converting JSON records to DataFrames and showing `st.info`/`st.error`.
- `inflation_dashboard/frontend/api_client.py` completed endpoint wrappers: `fetch_history`, `fetch_retailer_averages`, `fetch_movers`, and `fetch_coverage`, plus cached-call-compatible parameter builders for endpoint-specific params.
- `scripts/verify_streamlit_api_frontend.py` tightened final Phase 3 assertions: no forbidden CSV repository import, no direct use-case import/calls, endpoint strings/client methods for all four tabs, repeated retailer params, error/envelope checks, tab labels, and autocorrection controls.
- API endpoint usage produced by this plan: `/api/history`, `/api/retailer-averages`, `/api/movers`, and `/api/coverage` in addition to 03-01 `/api/inventory`.
